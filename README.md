### （1）题目意思与生僻概念解释

**题目意思：**
给出一个未排序的整数数组，要求找出其中**没有出现过的、最小的正整数**（正整数从 1 开始：1, 2, 3...）。
例如：数组 `[3, 4, -1, 1]` 中，出现了 1、3、4，缺失的最小正整数就是 2。
并且题目通常会附加极其苛刻的限制条件：时间复杂度必须是 $O(n)$，且只能使用 $O(1)$ 的额外常数空间。

**核心生僻概念解释：**
* **原地哈希（In-place Hashing） / 抽屉原理：** 既然只能用 $O(1)$ 的空间，就不能额外开辟一个哈希表去记录哪些数字出现过。核心思想是**把数组自身当成哈希表（抽屉）**。规则是：数字 `1` 应该放在下标 `0` 的位置，数字 `2` 应该放在下标 `1` 的位置……即数字 `x` 应该放在下标 `x - 1` 的位置。
* **双指针区间划分：**
    * **左指针 `l`：** 维护“已归位的有效正数区”。区间 `[0, l)` 里的数字绝对是完美的 `1, 2, 3...l`。
    * **右指针 `r`：** 维护“垃圾区”。区间 `[r, n)` 里的数字都是经过判定，对寻找当前连续正数序列**毫无帮助**的废数字。
    * **中间区间 `[l, r)`：** 尚未被检查、等待处理的盲区。

### （2）代码解题思路解析

代码的核心在于不断审视当前指针 `l` 所指的数字 `nums[l]`，并根据它的价值将其分为三类进行处理，直到盲区消失（`l == r`）：

1.  **场景一（完美归位）：** 如果 `nums[l]` 恰好等于 `l + 1`，说明这个数字正好在它该在的位置上。有效区向右扩张（`l += 1`）。
2.  **场景二（判定为垃圾）：** 遇到以下三种情况，直接将其视为“垃圾”，将其与盲区最右侧的数字（下标为 `r - 1`）交换，并将垃圾区向左扩张（`r -= 1`）：
    * `nums[l] <= l`：非正数（$\le 0$），或者是比当前 `l` 还要小的值。因为 `[0, l)` 已经排好了完美序列，再来一个更小的正数只可能是重复的无用数字。
    * `nums[l] > r`：数字大得离谱。当前盲区加有效区总共只剩下 `r` 个槽位，最多只能凑出从 1 到 `r` 的连续正数。一旦遇到大于 `r` 的数，它绝对插不进当前的完美序列中。
    * `nums[nums[l] - 1] == nums[l]`：要去的那个“抽屉”里，已经躺着一个和它一模一样的数字了（重复）。
3.  **场景三（有用但放错位置）：** 既不是当前位置的完美数字，也不是垃圾。说明它是一个有价值的数字，只是放错了地方。将其与它该去的位置（下标 `nums[l] - 1`）上的数字**交换**。注意，换完之后 `l` 指针**不移动**，因为从远方换过来的新数字还需要在下一轮继续被审视。

### （3）带有注释的代码框

```python
class Solution:
    def firstMissingPositive(self, nums: list[int]) -> int:
        # left 指针：有效区的右边界，[0, left) 内的元素保证是 1, 2, 3...
        left = 0
        # right 指针：垃圾区的左边界，[right, len(nums)) 内的元素被判定为无用
        right = len(nums)  
        # 当盲区 [left, right) 还存在时，继续循环处理
        while left < right:
            # 场景 1: 当前位置的数字恰好等于期望值（下标 left 对应数字 left + 1）
            if nums[left] == left + 1:
                left += 1  # 有效区向右扩张
            # 场景 2: 当前数字是垃圾 -> 负数/过小、过大、目标位置已有重复数字
            elif nums[left] <= left or nums[left] > right or nums[nums[left] - 1] == nums[left]:
                right -= 1  # 垃圾区向左扩张
                # 将被判定为垃圾的当前数字，与盲区最右侧的未知数字交换
                nums[left], nums[right] = nums[right], nums[left]
            # 场景 3: 数字有效且未归位 -> 将其送到它该去的下标 (nums[left] - 1)
            else:
                # 提前提取目标索引，防止 Python 多重赋值特性带来的覆盖 bug
                target_index = nums[left] - 1
                # 执行交换操作，注意 left 指针停在原地，等待下一轮审视被换过来的新数字
                nums[left], nums[target_index] = nums[target_index], nums[left]
        # 循环结束，有效区 [0, left) 存放了 1 到 left 的完美序列
        # 因此，缺失的最小正数必然是紧随其后的 left + 1
        return left + 1
```

### （4）每一行代码详细中文解释

1.  `class Solution:`
    * **解释：** 定义解题类。
2.  `def firstMissingPositive(self, nums: list[int]) -> int:`
    * **解释：** 定义方法，接收整数列表 `nums`，返回缺失的最小正数。
3.  `l = 0`
    * **解释：** 初始化左指针 `l`。代表目前已经找到了 0 个完美归位的连续正数。
4.  `r = len(nums)`
    * **解释：** 初始化右指针 `r`。初始状态下，整个数组的长度就是理论上能凑出的最大连续正数的极限值。
5.  `while l < r:`
    * **解释：** 只要左指针小于右指针，说明 `[l, r)` 之间还有没被检查过的元素，循环继续。
6.  `if nums[l] == l + 1:`
    * **解释：** 检查当前数字是否恰好匹配当前的“坑位”。（例如 `l=0` 时期待数字 `1`，`l=1` 时期待数字 `2`）。
7.  `l += 1`
    * **解释：** 匹配成功，说明有效序列成功延长了一格，左指针向右移动。
8.  `elif nums[l] <= l or nums[l] > r or nums[nums[l] - 1] == nums[l]:`
    * **解释：** 核心“垃圾”判定逻辑：
        * `nums[l] <= l`：包含了负数、0、以及已经收集过的较小正数。
        * `nums[l] > r`：数字超出了剩余可能凑出的极限范围。
        * `nums[nums[l] - 1] == nums[l]`：想把它送回下标 `nums[l]-1` 的老家，却发现老家已经有一个同样的数字占座了。
9.  `r -= 1`
    * **解释：** 既然确认当前数字是没用的垃圾，那就把可用区间的上限压缩一格（右边界左移）。
10. `nums[l], nums[r] = nums[r], nums[l]`
    * **解释：** 将这个垃圾数字丢到最新划定的垃圾区边界 `r` 的位置。同时，把原本位于 `r` 位置那个还没检查过的盲区数字换到 `l` 位置来。
11. `else:`
    * **解释：** 排除上述所有情况，说明这个数字是个“好人”，应该去填补前面还没填上的某个坑位。
12. `target_idx = nums[l] - 1`
    * **解释：** 计算这个好人该回的老家下标。在 Python 中，如果直接写 `nums[l], nums[nums[l]-1] = ...`，由于左到右的赋值计算顺序，`nums[l]` 先被改变后，后面的索引 `nums[l]-1` 也会跟着突变，导致位置错乱。必须单独提取。
13. `nums[l], nums[target_idx] = nums[target_idx], nums[l]`
    * **解释：** 将当前数字送回老家，同时把老家原本占位的另一个无辜数字换回 `l` 的位置。由于换回来的新数字还没被检查，所以下一轮 `while` 循环将继续在不动 `l` 的情况下审视它。
14. `return l + 1`
    * **解释：** 盲区耗尽。区间 `[0, l)` 被完美数字填满。自然而然，缺失的下一个数就是 `l + 1`。

### （5）具体数值算例与变量变化过程

**设定算例：** `nums = [3, 4, -1, 1]`，初始长度 `n = 4`。

| 轮次 | 变量现状 | 判定逻辑 | 数组交换状态 | 对应核心代码 |
| :--- | :--- | :--- | :--- | :--- |
| **初始** | `l=0, r=4` <br> `nums[0] = 3` | - | `[3, 4, -1, 1]` | `l = 0`, `r = 4` |
| **1** | `l=0, r=4` <br> `nums[0] = 3` | 3 应该放在下标 2。<br> 满足`else`场景，与`nums[2]`交换。 | `[-1, 4, 3, 1]` | `nums[l], nums[target] =...` |
| **2** | `l=0, r=4` <br> `nums[0] = -1` | 新换来的 -1 $\le$ 0，是垃圾。<br>与 `nums[r-1]`(即 `nums[3]`) 交换。 | `[1, 4, 3, -1]` | `r -= 1` <br> `nums[l], nums[r] = ...` |
| **3** | `l=0, r=3` <br> `nums[0] = 1` | 新换来的 1 刚好等于 `l+1`，完美归位！<br>左指针移动。 | `[1, 4, 3, -1]` | `l += 1` |
| **4** | `l=1, r=3` <br> `nums[1] = 4` | 当前坑位需要 2，来了 4。<br>注意：此时 `r=3`。`4 > 3`，超出极限范围，判定为垃圾。<br>与 `nums[r-1]`(即 `nums[2]`) 交换。 | `[1, 3, 4, -1]` | `r -= 1` <br> `nums[l], nums[r] = ...` |
| **5** | `l=1, r=2` <br> `nums[1] = 3` | 当前坑位需要 2，来了 3。<br>此时 `r=2`。`3 > 2`，再次超出极限，判定为垃圾。<br>与 `nums[r-1]`(即 `nums[1]`) 交换自身。 | `[1, 3, 4, -1]` | `r -= 1` <br> `nums[l], nums[r] = ...` |
| **结束** | `l=1, r=1` | `l < r` 不成立，循环结束。 | `[1, 3, 4, -1]` | `return l + 1` |

最终输出：`l + 1` 即 `1 + 1 = 2`。
